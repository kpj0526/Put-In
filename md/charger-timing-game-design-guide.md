# 충전기 타이밍 게임 MVP 디자인 가이드

## 1. 디자인 방향

충전기 타이밍 게임은 짧은 반복 플레이가 핵심인 캐주얼 아케이드 게임이다.

디자인은 "밤에 휴대폰을 충전기에 꽂는 순간"을 과장한 전자기기 아케이드 톤으로 잡는다. 화면 중앙의 콘센트와 좌우로 움직이는 충전기가 즉시 눈에 들어와야 하며, 결과 화면에서는 배터리가 차오르는 만족감이 강하게 보여야 한다.

## 2. 키워드

- 전기
- 충전
- 타이밍
- 네온
- 배터리
- 짧은 재도전
- 정확도 경쟁

## 3. 컬러 팔레트

| 용도 | 색상 | Hex |
| --- | --- | --- |
| 배경 기본 | Deep Graphite | `#15171C` |
| 배경 보조 | Charcoal | `#222832` |
| 주요 강조 | Electric Mint | `#31F5A6` |
| 보조 강조 | Signal Yellow | `#FFD84D` |
| 위험/실패 | Error Coral | `#FF5A6A` |
| 정보 | Cool Cyan | `#58D7FF` |
| 텍스트 기본 | Soft White | `#F5F7FA` |
| 텍스트 보조 | Muted Steel | `#9BA7B4` |
| 경계선 | Dim Line | `#39414F` |

### 사용 기준

- 전체 배경은 어두운 전자기기 느낌을 유지한다.
- 성공, 충전, 진행 상태는 `Electric Mint`를 사용한다.
- 100% 또는 높은 정확도는 `Signal Yellow`를 함께 사용해 보상감을 준다.
- 실패나 낮은 정확도는 `Error Coral`을 사용한다.
- 보라색 계열 그라디언트는 사용하지 않는다.

## 4. 타이포그래피

### 권장 폰트

| 용도 | 폰트 |
| --- | --- |
| 영문 숫자 점수 | `Bebas Neue` 또는 `Barlow Condensed` |
| 한글 UI | `Pretendard` |
| 보조 텍스트 | `Pretendard` |

### 사용 기준

- 정확도 숫자는 크게 보여준다.
- 버튼 텍스트는 짧고 명확하게 쓴다.
- 게임 중 화면에는 설명 문구를 최소화한다.
- 리더보드와 폼 화면은 가독성을 우선한다.

## 5. 화면 구성

### 5.1 인증 화면

### 목적

이메일 로그인 또는 회원가입을 빠르게 완료하게 한다.

### 구성 요소

- 게임 로고 또는 타이틀
- 이메일 입력
- 비밀번호 입력
- 닉네임 입력: 회원가입 화면에서만 표시
- 로그인 버튼
- 회원가입 전환 버튼
- 에러 메시지

### 디자인 기준

- 폼은 화면 중앙에 배치한다.
- 배경에는 희미한 배터리 게이지 패턴을 깔 수 있다.
- 입력창은 어두운 배경 위에 밝은 경계선으로 구분한다.
- 비밀번호 조건은 회원가입 화면에서만 작게 표시한다.

### 5.2 시작 화면

### 목적

게임을 바로 시작하게 한다.

### 구성 요소

- 게임 타이틀
- 현재 로그인 닉네임
- 시작 버튼
- 리더보드 버튼
- 로그아웃 버튼

### 디자인 기준

- 시작 버튼은 화면에서 가장 강하게 보여야 한다.
- 타이틀 주변에는 약한 전류 라인 애니메이션을 사용할 수 있다.
- 불필요한 설명 문구는 넣지 않는다.

### 5.3 게임 화면

### 목적

충전기와 콘센트의 위치 관계를 즉시 판단하게 한다.

### 구성 요소

- 중앙 콘센트
- 좌우로 움직이는 충전기
- 배터리 UI
- 현재 상태 텍스트

### 레이아웃

```text
┌─────────────────────────┐
│ 배터리 1%               │
│                         │
│          콘센트          │
│     ← 충전기 이동 →      │
│                         │
│      화면 클릭/터치      │
└─────────────────────────┘
```

### 디자인 기준

- 콘센트는 화면 중앙에 고정한다.
- 충전기는 콘센트와 같은 수평선 위에서 움직인다.
- 이동 범위는 화면 폭보다 작게 제한해 조작 목표가 명확해야 한다.
- 충전기에는 전선 꼬리나 작은 그림자를 넣어 물체감이 나게 한다.
- 입력 후에는 충전기가 즉시 멈추고, 결과 연출로 자연스럽게 넘어간다.

### 5.4 결과 화면

### 목적

정확도 결과와 재도전 욕구를 강하게 보여준다.

### 구성 요소

- 정확도 %
- 판정 문구
- 배터리 충전 애니메이션
- 기록 등록 상태
- 현재 순위
- 다시하기 버튼
- 리더보드 버튼

### 디자인 기준

- 정확도 숫자는 화면에서 가장 크게 표시한다.
- 배터리는 정확도만큼 채워지는 애니메이션을 적용한다.
- 90% 이상은 밝은 충전 효과를 강하게 보여준다.
- 40% 미만은 약한 스파크 또는 실패 색상을 사용한다.

### 5.5 리더보드 화면

### 목적

상위 기록과 내 최근 기록을 빠르게 비교하게 한다.

### 구성 요소

- 상위 10개 기록
- 순위
- 닉네임
- 정확도
- 판정
- 기록 시간
- 닫기 또는 뒤로가기 버튼

### 디자인 기준

- 1~3위는 시각적으로 구분한다.
- 이메일은 표시하지 않는다.
- 내 최근 기록은 별도 강조 색상으로 표시한다.
- 표 형태를 기본으로 하되 모바일에서는 리스트 형태로 바꾼다.

## 6. 주요 컴포넌트

### 버튼

| 종류 | 사용 위치 | 스타일 |
| --- | --- | --- |
| Primary | 시작, 다시하기, 로그인 | Electric Mint 배경 |
| Secondary | 리더보드, 회원가입 전환 | 투명 배경 + Dim Line 경계 |
| Danger | 로그아웃 | Error Coral 텍스트 또는 경계 |

### 입력창

- 높이는 최소 44px 이상으로 한다.
- 모바일 터치를 고려해 충분한 여백을 둔다.
- 에러 상태에서는 `Error Coral` 경계선을 사용한다.

### 배터리 UI

- 시작 시 1%로 표시한다.
- 결과 화면에서는 정확도만큼 채워진다.
- 90% 이상은 `Electric Mint`와 `Signal Yellow`를 함께 사용한다.
- 40% 미만은 `Error Coral` 계열로 표시한다.

### 콘센트

- 중앙 목표물로 명확하게 보여야 한다.
- 원형 또는 사각 콘센트 플레이트를 사용할 수 있다.
- 중심점을 시각적으로 알 수 있게 작은 빛 또는 홈을 둔다.

### 충전기

- 화면에서 좌우로 움직이는 주 오브젝트다.
- 콘센트와 맞닿는 끝부분이 정확히 보이게 한다.
- 멈췄을 때 콘센트 중심과 충전기 중심이 비교 가능해야 한다.

## 7. 판정별 연출

| 판정 | 색상 | 연출 |
| --- | --- | --- |
| PERFECT_CHARGE | Signal Yellow + Electric Mint | 강한 플래시, 배터리 즉시 100% |
| GREAT | Electric Mint | 부드러운 충전 파동 |
| GOOD | Cool Cyan | 짧은 충전 효과 |
| WEAK | Signal Yellow | 약한 점멸 |
| BAD | Error Coral | 짧은 오류 진동 |
| MISS | Error Coral | 충전 실패 효과 |

## 8. 애니메이션 기준

- 충전기 이동은 일정 속도로 부드럽게 움직인다.
- 화면 끝에 닿으면 즉시 방향을 반전한다.
- 입력 순간 충전기는 딜레이 없이 멈춘다.
- 결과 배터리 충전 애니메이션은 0.6~1.0초 안에 끝낸다.
- 버튼 hover는 색상 변화와 약한 밝기 변화 정도로 제한한다.
- 모바일에서는 과한 흔들림 애니메이션을 줄인다.

## 9. 반응형 기준

### Desktop

- 게임 영역 최대 폭은 720px 정도로 제한한다.
- 좌우 여백을 넉넉히 두고 중앙 집중형 화면을 유지한다.
- 리더보드는 표 형태로 표시한다.

### Mobile

- 게임 영역은 화면 너비를 거의 다 사용한다.
- 버튼과 입력창 높이는 최소 44px 이상으로 한다.
- 리더보드는 카드형 리스트로 표시한다.
- 조작 영역을 넓게 잡아 화면 어디를 터치해도 입력되게 한다.

## 10. 접근성 기준

- 키보드 `Enter`, `Space` 입력을 지원한다.
- 버튼과 입력창은 focus 상태가 명확해야 한다.
- 색상만으로 성공/실패를 구분하지 않고 판정 문구를 함께 표시한다.
- 에러 메시지는 입력창 근처에 표시한다.

## 11. 구현 메모

- 첫 MVP에서는 CSS와 간단한 SVG 또는 div 기반 그래픽으로 구현해도 충분하다.
- 충전기와 콘센트는 나중에 이미지 에셋으로 교체할 수 있게 컴포넌트를 분리한다.
- 사운드는 버튼 클릭, 성공, 실패 세 종류부터 적용한다.
- 리더보드 조회 실패 시 게임 플레이 자체는 막지 않는다.
- 로그인되지 않은 사용자는 게임 시작 전에 로그인 화면으로 보낸다.
